## Conformance Workflow

On session start, a hook runs `bin/run spec check "$PROJECT_DIR" --format hook` producing:

- **`Core: READY`** — all specifications conformant, no action needed
- **`Core: VIOLATIONS (CORE-SPEC-001, CORE-SPEC-002)`** — listed specs have non-conformances

To remediate, invoke the corresponding skill: `/conformance-testing-core-spec-001`.
The skill forks a remediation-agent that parses violations, applies fixes, and verifies convergence to READY.

## CLI Commands

```bash
run spec list                              # List all specifications
run spec info CORE-SPEC-001                # Show specification details
run spec check .                           # Check all specs for conformance
run spec check . --spec CORE-SPEC-001      # Check single spec
run spec check . --format hook             # Hook-friendly output
run hooks inspect                          # Inspect hook traces from latest session
run hooks inspect --session <id>           # Inspect specific session
run hooks inspect --event SessionStart     # Filter by event
```
